# Example Skill（雛形）

このファイルは `npx skills add` で配布可能なスキルの最小テンプレート。
新しいスキルを作るときは、このディレクトリを複製してリネームし、中身を書き換える。

> 配置ルール（カタログ型）: `skills/<category>/<skill-name>/SKILL.md`
> `name` は小文字・ハイフン区切り。ディレクトリ名と一致させると分かりやすい。

## 目的

このスキルが何を達成するためのものかを1〜2文で書く。
「何を扱い／何を扱わないか」の境界も明示すると誤発火を防げる。

## 進め方

1. 最初のステップ（例: ユーザーの依頼から対象を特定する）
2. 次のステップ（例: 必要な情報を集める）
3. 仕上げ（例: 結果を整形して返す）

## 出力形式

最終的にユーザーへ返す成果物の形を定義する（見出し構成・表・コードブロック等）。

## 注意点

- frontmatter の `name` / `description` は必須。`description` の質が発火精度を左右する。
- スキルが大きくなる場合は、このディレクトリに参照ファイル（`reference.md` 等）を追加し、本文から参照させる。
